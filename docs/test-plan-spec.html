1. Scope & Packages

The claude-config-manager (CCM) project is a Turborepo monorepo containing five packages that together provide configuration management for Claude Code via a CLI, a web dashboard, an MCP server, shared business logic, and TypeScript type definitions.

Package Description Key Exports / Entry Points Runtime
@ccm/core Business logic layer PluginManager, McpManager, ProfileManager, ConfigManager, SkillScanner Node.js
@ccm/dashboard Next.js web UI Pages, API routes, React components Browser + Node.js
@ccm/mcp MCP server exposing 19 tools JSON-RPC server, tool handlers Node.js
@ccm/cli Command-line interface Commands: start, list, profile, export, import, mcp-server Node.js
@ccm/types Shared TypeScript type definitions, type guards and validators Interfaces, enums, type guards, validators Build-time + runtime

14. Risks & Mitigations

RiskImpactLikelihoodMitigation
File system race conditions on concurrent writes High Medium Atomic write-rename pattern; integration test I-12 validates
Cross-platform path separator issues Medium High Use path.join() everywhere; CI matrix includes Windows
MCP server hangs under malformed input High Low Protocol tests P-03, P-04; input validation at handler entry
Dashboard XSS via user-provided plugin/profile names High Low React auto-escaping; explicit security tests S-08, S-09
Credential leakage in profile exports Critical Medium Default exclude; security tests S-01, S-02; code review gate
Flaky E2E tests in CI Low High Playwright retries (2x in CI); trace on first retry; stable selectors
Turborepo cache invalidation hiding real failures Medium Low Nightly full clean build; --force flag on release CI
Large plugin counts degrading performance Medium Medium Performance tests PF-03, PF-04 with 50+ / 200+ items

D. Exit Criteria

A release is considered test-complete when all of the following are met:

  • All P0 test cases pass across all CI matrix combinations
  • All P1 test cases pass on the primary environment (Node 20, Ubuntu)
  • Coverage thresholds met: 80% lines for @ccm/core, 70% for others
  • Zero known security test failures (S-01 through S-10)
  • E2E test suite passes on Chromium and Firefox
  • No performance regressions beyond defined thresholds
  • All lint and type-check gates pass
